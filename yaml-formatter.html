What is a YAML formatter?

      
A YAML formatter takes YAML with inconsistent or messy indentation and rewrites it with clean, consistent 2-space indentation, the convention used by nearly every style guide and linter. Because YAML uses whitespace to express structure, a single misplaced space can silently change what a file means or break it outright — this tool parses your YAML into a real data structure first, so formatting only succeeds once the structure has been correctly understood.

      
How to use it

      
1. Paste your YAML into the box above. 2. Click Format to reformat it with consistent 2-space indentation, YAML → JSON to convert it into the equivalent JSON, or Validate to check it for structural errors. 3. Click Copy to grab the result. If your YAML has a malformed indentation level, the status line reports which line it happened on.

      
What does this formatter support?

      
This tool handles the YAML you'll find in most real-world config files: nested key-value mappings, sequences written with a leading -, single- and double-quoted strings, unquoted plain strings, integers and floats, booleans (true/false, yes/no), and null/~. It does not implement the full YAML 1.2 specification — anchors and aliases (&/*), multi-document streams (---), flow collections ({a: 1}/[1,2] are supported as scalars-in-brackets but not deeply parsed), and block scalar indicators (|/>) are outside its scope. For the common case of hand-written config YAML, it correctly reconstructs the nested structure.

      
Is my data safe?

      
Yes. Parsing, formatting, and conversion all happen entirely in your browser using plain JavaScript. Nothing you paste — including secrets or config values — is ever sent to a server.
