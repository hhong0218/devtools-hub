What is a JSON to TypeScript converter?

      
A JSON to TypeScript converter reads a sample of JSON — typically an API response or a config file — and generates the matching TypeScript interface (or type) declarations. Instead of hand-typing every property and guessing its type, you paste the JSON and get a ready-to-use static type: nested objects become their own interfaces, arrays are inferred to an element type, and mixed values collapse into unions. It is the fastest way to give an untyped API a typed shape.

      
How to use it

      
1. Paste a JSON object or array into the input box (click Sample to see the expected shape). 2. Optionally set the Root name for the top-level type and pick interface or type output. 3. Press Generate, then Copy the result into your project. Toggle null → optional to turn null values into optional properties, or leave it off to keep an explicit null in the type.

      
How are nested objects and arrays handled?

      
Nested objects are extracted into their own named interfaces, so a user.address object becomes an Address interface referenced from the parent. Arrays are inferred by merging the types of their elements: an array of objects produces one interface describing the union of all keys seen (keys missing from some elements become optional), and an array of primitives becomes a typed array such as string[] or (string | number)[]. Empty arrays fall back to unknown[], and empty objects become Record<string, unknown> so nothing is silently dropped.

      
Is my data safe?

      
Yes. The conversion runs entirely in your browser using plain JavaScript and the built-in JSON.parse engine. Nothing you paste is uploaded to a server, making it safe for internal API responses and private schemas.

      
Frequently asked questions

      
Should I paste a single object or an array? Either works. A single object generates one root interface; an array generates a root interface for its elements plus a root type alias like type Root = RootItem[].

      
What happens to null values? With null → optional on, a property that is null in the sample becomes optional (name?: string if seen as a string elsewhere, or name?: null if only ever null). Turn the toggle off to emit an explicit null in the type instead.

      
Can it produce type aliases instead of interfaces? Yes — switch the Output selector to type and every generated declaration uses type X = { … } syntax.
