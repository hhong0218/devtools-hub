URL Encoding Explained

URL encoding — also called percent-encoding — is how text that contains reserved or unsafe characters gets safely placed inside a web address. It's the reason a space turns into %20 and an ampersand inside a value becomes %26. This guide explains what gets encoded, why, and the difference between the two JavaScript functions developers reach for.

Why URLs need encoding

A URL has a strict grammar. Characters like /, ?, &, =, and # have structural meaning — they separate the path from the query string, one parameter from the next, and so on. If a value you want to send contains one of those characters, the browser or server would misread where the value ends.

URL encoding solves this by replacing any unsafe byte with a percent sign followed by its two-digit hexadecimal value. So a literal ampersand inside a search term becomes %26, and the parser no longer confuses it with a parameter separator.

Input : name=Tom & Jerry
Encoded: name=Tom%20%26%20Jerry

Which characters get encoded

Characters fall into three buckets:

GroupExamplesEncoded?
UnreservedA–Z a–z 0–9 - _ . ~Never
Reserved: / ? # [ ] @ ! $ & ' ( ) * + , ; =When used as data, not structure
Other / non-ASCIIspaces, é, 中, emojiAlways (as UTF-8 bytes)

Non-ASCII characters are first turned into their UTF-8 bytes, then each byte is percent-encoded. That's why a single emoji can expand into several %XX sequences.

encodeURI vs encodeURIComponent

JavaScript gives you two functions, and choosing the wrong one is a classic bug.

  • encodeURI() — for an entire URL. It leaves structural characters (: / ? & =) alone so the URL stays valid.
  • encodeURIComponent() — for a single piece of data, like one query-string value. It escapes &, =, / and the rest so they can't break the URL's structure.
⚠️ Rule of thumb: building a value to drop into a parameter? Use encodeURIComponent. Encoding a full link you already assembled? Use encodeURI. When unsure, encode the individual values.

Spaces: %20 vs +

You'll see spaces encoded two ways. In the path and most contexts a space is %20. In application/x-www-form-urlencoded data — the classic HTML form submission — a space is often a +. Both are valid in their place; decoders that follow the form rules treat + as a space.

Frequently asked questions

Why does a space become %20? Space is byte 0x20 and isn't allowed raw in a URL, so it's percent-encoded to %20.

encodeURI or encodeURIComponent? Use encodeURIComponent for individual values; encodeURI for a whole URL.

Is encoding the same as encryption? No — it's reversible formatting with zero secrecy.
