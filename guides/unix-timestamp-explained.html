What Is a Unix Timestamp?

A Unix timestamp — also called epoch time or POSIX time — is a single number that represents a moment in time: the count of seconds that have elapsed since midnight UTC on 1 January 1970. It's how databases, APIs, log files, and programming languages store time under the hood. This guide explains where it comes from, the seconds-vs-milliseconds trap, and the famous Year 2038 problem.

One number, no timezone

Instead of storing a date as "June 25, 2026 3:42 PM in Seoul," computers store a plain integer like 1782000000. The beauty of this is that the number is timezone-independent — it refers to the same instant everywhere on Earth. You only apply a timezone when you display it to a human.

0            → 1970-01-01 00:00:00 UTC (the epoch)
1000000000   → 2001-09-09 01:46:40 UTC
1782000000   → 2026-06-...  (a moment in 2026)

Why 1970?

When Unix was being built in the early 1970s, its authors needed a fixed zero point to count from. They picked midnight UTC on 1 January 1970 — a clean, recent round date. That instant became known as the epoch, and counting seconds from it turned out to be simple, compact, and easy to do arithmetic on.

Seconds vs milliseconds — a common bug

This trips up developers constantly. Classic Unix timestamps are in seconds, but many languages — JavaScript especially — work in milliseconds.

ValueDigitsUnit
178200000010seconds
178200000000013milliseconds
💡 Quick check: a 10-digit number is seconds, a 13-digit number is milliseconds. Mixing them up sends a date to either 1970 or the year 50,000+. Date.now() in JavaScript returns milliseconds.

The Year 2038 problem

Many older systems store the timestamp in a signed 32-bit integer. The largest value that holds is 2,147,483,647 — which, counted as seconds from 1970, lands on 19 January 2038. One second later the counter overflows and wraps to a large negative number, throwing dates back to 1901. The fix is to store the value in a 64-bit integer, which pushes the limit hundreds of billions of years out. Most modern operating systems and languages already use 64-bit time.

Frequently asked questions

Why does Unix time start in 1970? The Unix authors chose midnight UTC on 1 Jan 1970 as a convenient fixed reference, the epoch.

Seconds or milliseconds? Traditionally seconds (10 digits); JavaScript uses milliseconds (13 digits).

What's the Year 2038 problem? 32-bit timestamps overflow on 19 Jan 2038; the fix is 64-bit storage.
