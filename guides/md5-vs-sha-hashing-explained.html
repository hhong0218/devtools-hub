MD5 vs SHA Hashing Explained

Hash functions turn any input — a word, a file, a gigabyte of data — into a fixed-length fingerprint. MD5, SHA-1, and SHA-256 are the names you meet most often, and they are not interchangeable. This guide explains what a hash is, why MD5 and SHA-1 are considered broken, and which algorithm to reach for depending on the job.

What a hash function does

A cryptographic hash takes input of any size and returns a fixed-size string. The same input always yields the same output, but even a one-character change produces a completely different result:

md5("hello")  = 5d41402abc4b2a76b9719d911017c592
md5("hellp")  = d96b9c2d3c8f8a6e... (totally different)

Three properties make a hash cryptographic: it's a one-way function (you can't reverse it to recover the input), it's deterministic, and it should be infeasible to find two inputs with the same output. That last property — collision resistance — is exactly where MD5 fails.

Why MD5 is broken

MD5 was designed in 1991 and was the default checksum for years. The problem is collisions: researchers can deliberately construct two different files that produce the same MD5 hash. That destroys its value for integrity checks — a malicious file could be swapped in while matching the expected hash. SHA-1 fell to a practical collision attack (known as "SHAttered") in 2017, putting it in the same category.

⚠️ Treat MD5 and SHA-1 as "checksum-only" tools. They can catch accidental corruption, but they cannot defend against a motivated attacker.

Which one should you use?

  • Integrity / digital signatures / anything security-sensitive — use SHA-256 (or SHA-512). These are the current standard.
  • Fast non-security checksums — detecting an accidentally corrupted download where no attacker is involved — MD5 is acceptable, though SHA-256 is barely slower on modern hardware.
  • Passwords — do not use any of these directly. General hashes are too fast, which helps brute-force attacks. Use bcrypt, scrypt, or Argon2, which are deliberately slow and salted.

Frequently asked questions

Is MD5 still safe? Only as a non-security checksum. It's broken for anything where an attacker could tamper with data.

MD5 vs SHA-256? SHA-256 produces a longer hash and is collision-resistant; prefer it whenever integrity matters.

Hash passwords with SHA-256? No — use bcrypt, scrypt, or Argon2 instead.
