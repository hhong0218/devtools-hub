Regex Cheatsheet

Regular expressions are a compact language for matching patterns in text. They power search-and-replace, input validation, log parsing, and routing. The syntax can look cryptic, but the core building blocks are few. This cheatsheet collects the pieces you reach for most, with examples you can drop straight into a tester.

Anchors and boundaries

TokenMatches
^Start of the string (or line, in multiline mode)
$End of the string (or line)
\bA word boundary, e.g. between a letter and a space
\BA non-word-boundary

Character classes

TokenMatches
.Any character except newline
\d / \DA digit / a non-digit
\w / \WA word character ([A-Za-z0-9_]) / non-word
\s / \SWhitespace / non-whitespace
[abc]Any one of a, b, or c
[^abc]Any character except a, b, or c
[a-z]Any character in the range a to z

Quantifiers

TokenMeaning
*Zero or more
+One or more
?Zero or one (optional)
{3}Exactly 3
{2,5}Between 2 and 5
{2,}2 or more
*? +?Lazy versions — match as few as possible

Groups and alternation

  • (abc) — a capturing group you can reference later
  • (?:abc) — a non-capturing group (groups without capturing)
  • (?<year>\d{4}) — a named capturing group
  • cat|dog — alternation: matches cat or dog
  • (?=...) / (?!...) — positive / negative lookahead

Ready-to-use patterns

GoalPattern
Simple email^[^@\s]+@[^@\s]+\.[^@\s]+$
ISO date (YYYY-MM-DD)^\d{4}-\d{2}-\d{2}$
Hex color^#?([0-9a-fA-F]{3}|[0-9a-fA-F]{6})$
US phone (loose)\(?\d{3}\)?[-.\s]?\d{3}[-.\s]?\d{4}
Trim whitespace^\s+|\s+$
Whole word "error"\berror\b
Treat patterns like the email one as "good enough" filters, not strict validators. Email and URL grammars are notoriously complex; a forgiving pattern plus a real send/confirmation step beats an over-strict regex that rejects valid input.

Test as you go

The fastest way to learn regex is to iterate: tweak the pattern, watch the matches light up, repeat.
